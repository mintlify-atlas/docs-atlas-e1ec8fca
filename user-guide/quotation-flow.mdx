## Overview

The quotation system guides you through a three-step process to calculate personalized healthcare plan pricing based on your family composition and ages. The system dynamically adapts based on your selections to provide accurate pricing from multiple healthcare providers.

<Steps>
  <Step title="Select Family Type">
    In the first step, you choose the family composition that matches your situation. This determines which information fields you'll need to complete in the next step.

    ### Available Options

    <CardGroup cols={2}>
      <Card title="Para mí" icon="user">
        Individual plan for one person
      </Card>
      <Card title="Para mí y mi pareja" icon="user-friends">
        Family plan for 2 people (couple)
      </Card>
      <Card title="Para mí y mis hijos" icon="baby">
        Family plan with children included (single parent)
      </Card>
      <Card title="Para mi familia completa" icon="home">
        Complete family plan (couple + children)
      </Card>
    </CardGroup>

    ### How It Works

    Click on any option card to select it. The selected option will be highlighted and the "Continuar" button will become active.

    ```javascript
    // Selection stored in selectedOption variable

    ```

    <Note>
      Your selection determines the form fields shown in Step 2. For example, selecting "Para mi familia completa" will show fields for both partner age and children ages.
    </Note>
  </Step>

  <Step title="Enter Family Member Details">


    <Info>
      The form validates that all required fields are completed before allowing you to proceed. Age fields must contain valid numeric values.
    </Info>

    ### Age Groups Matter

    Different providers use different age brackets for pricing:

    - **OMINT**: 0-25, 26-35, 36-54, 55-59, 60+
    - **SWISS MEDICAL**: ≤65, >65
    - **MEDIFE**: 0-29, 30-39, 40-49, 50-59, 60+
    - **ACTIVA SALUD**: Most plans use single pricing (≤100)
    - **OSPADEP SALUD**: 18-27, 28-35, 36-40, 41-45, 46-50, 51-55, 56-64, 65+

    The system automatically places each family member in the correct age bracket for each provider.
  </Step>

    ### Price Calculation Process

    Behind the scenes, the system:

    1. **Loads pricing data** from `precios-data.json`
    2. **Determines age brackets** for each family member
    3. **Applies provider-specific pricing logic**:
       - OMINT: Special rates for children (hijo1Menor, hijo2MasMenores)
       - SWISS MEDICAL: All members charged as adults (no child discounts)
       - MEDIFE: Complex structure with individual/matrimonio/hijos pricing
       - OSPADEP SALUD: Different pricing for afiliados vs particulares
    4. **Calculates totals** and displays results

    ```javascript
    // Example calculation for OMINT structure_compleja
    function calcularPrecioFinalOMINT(plan, composicion, edadTitular, edadPareja) {
      let precioTotal = 0;
      
      // 1. Add titular price
      precioTotal += plan.preciosPorEdad[grupoEtario].adultoConyugue;
      
      // 2. Add partner price (if applicable)
      if (composicion.tienePareja) {
        precioTotal += plan.preciosPorEdad[grupoPareja].adultoConyugue;
      }
      
      // 3. Add children with special rates
      if (todosLosHijos.length > 0) {
        precioTotal += plan.preciosPorEdad[grupoEtario].hijo1Menor;
        if (todosLosHijos.length > 1) {
          precioTotal += plan.preciosPorEdad[grupoEtario].hijo2MasMenores * (hijosAdicionales);
        }
      }
      
      return Math.round(precioTotal);
    }
    ```

    <Note>
      Prices shown are monthly premiums. All calculations include the most recent updates from the external pricing file.
    </Note>

## External Pricing System

The quotation system loads pricing from an external JSON file (`precios-data.json`) for easy updates:

```javascript
async function cargarPreciosExternos() {
  try {
    const response = await fetch('precios-data.json');
    if (response.ok) {
      const data = await response.json();
      prestadoresData = data.prestadores;
      preciosExternosDisponibles = true;
      return true;
    }
  } catch (error) {
    // Falls back to internal backup pricing
    return false;
  }
}
```

<Info>
  The system includes fallback pricing (respaldo) from September 2025 in case the external file cannot be loaded.
</Info>
