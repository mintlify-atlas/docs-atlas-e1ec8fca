## Overview

The `CONFIG` object centralizes all application settings including security, roles, sync intervals, and UI configuration.

---

## Access

```javascript
window.APP_CONFIG // Global configuration object
```

The configuration is automatically adjusted based on `ENVIRONMENT` setting.

---

## Configuration Fields

### Environment Settings

<ParamField path="ENVIRONMENT" type="string" default="development">
  Runtime environment: `development` or `production`
</ParamField>

**Example:**

```javascript
if (APP_CONFIG.ENVIRONMENT === 'production') {
  console.log('Running in production mode');
}
```

---

### Security Configuration

<ParamField path="ADMIN_PASSWORD_HASH" type="string">
  SHA-256 hash of admin password (production only)
  
  Default: `a665a45920422f9d417e4867efdc4fb8a04a1f3fff1fa07e998e86f7f7a27ae3`
</ParamField>

<ParamField path="SUPER_ADMIN_PASSWORD" type="string">
  Super administrator password
  
  Default: `superadmin2024`
</ParamField>

<ParamField path="SESSION_TIMEOUT" type="number" default="30">
  Session timeout in minutes
  
  - Production: 30 minutes
  - Development: 120 minutes
</ParamField>

<ParamField path="MAX_LOGIN_ATTEMPTS" type="number" default="3">
  Maximum failed login attempts before lockout
  
  - Production: 3 attempts
  - Development: 10 attempts
</ParamField>

---

### User Roles

<ParamField path="USER_ROLES" type="object">
  Defines available user roles
</ParamField>

**Structure:**

```javascript
USER_ROLES: {
  SUPER_ADMIN: 'super_admin',
  ADMIN: 'admin',
  USER: 'user'
}
```

**Usage:**

```javascript
if (currentUser.role === APP_CONFIG.USER_ROLES.SUPER_ADMIN) {
  // Super admin access
}
```

### Synchronization Settings

<ParamField path="SYNC_INTERVAL" type="number" default="5000">
  Auto-sync interval in milliseconds (5 seconds)
</ParamField>

<ParamField path="AUTO_SYNC" type="boolean" default="true">
  Enable automatic synchronization
</ParamField>

**Example:**

```javascript
if (APP_CONFIG.AUTO_SYNC) {
  setInterval(() => {
    syncData();
  }, APP_CONFIG.SYNC_INTERVAL);
}
```

## getConfig() Function

The `getConfig()` function returns environment-specific configuration.

```javascript
function getConfig()
```

**Behavior:**

- **Development Mode:**
  - Disables password hashing (`ADMIN_PASSWORD_HASH: null`)
  - Increases session timeout to 120 minutes
  - Allows 10 login attempts
  
- **Production Mode:**
  - Returns default CONFIG values
  - Enforces strict security settings

**Example:**

```javascript
window.APP_CONFIG = getConfig();

if (APP_CONFIG.ENVIRONMENT === 'development') {
  console.log('Development settings applied');
  console.log('Session timeout:', APP_CONFIG.SESSION_TIMEOUT, 'minutes');
}
```

## Complete Configuration Example

```javascript
// Access configuration
const config = window.APP_CONFIG;

// Security
if (config.SESSION_TIMEOUT < 60) {
  console.warn('Short session timeout configured');
}

// Roles
const isSuperAdmin = user.role === config.USER_ROLES.SUPER_ADMIN;

// Sync
if (config.AUTO_SYNC) {
  startAutoSync(config.SYNC_INTERVAL);
}

// Audit
if (config.LOG_ALL_ACTIONS) {
  logActivity('APP_STARTED');
}

// Pricing
const price = calculatePrice();
if (price < config.MIN_PRICE || price > config.MAX_PRICE) {
  console.error('Price out of bounds');
}

// UI
document.querySelector('.logo').src = config.LOGO_URL;
document.documentElement.style.setProperty('--theme-color', config.THEME_COLOR);
```

## Environment Variables

To switch between environments, modify the `ENVIRONMENT` field in `config.js`:

```javascript
const CONFIG = {
  ENVIRONMENT: 'production', // Change to 'development' for dev mode
  // ... rest of config
};
```
