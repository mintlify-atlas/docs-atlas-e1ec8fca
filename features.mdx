## Overview

Cotizador Online OSPADEP offers a complete healthcare quotation ecosystem with features for both end-users and administrators.

## Core Quotation Features

### Multi-Step Quotation Flow

<Info>
**Current Version:** 2026-02 (February 2026)
**Last Updated:** 2026-01-16

**Benefits:**
- Update prices without code changes
- Version tracking with change history
- Easy rollback to previous versions
- Supports multiple pricing structures

### LocalStorage Architecture

The system uses browser localStorage for:

| Key | Purpose | Size Limit |
|-----|---------|------------|
| `ospadep_users` | User accounts and permissions | ~5MB |
| `ospadep_active_session` | Current session data | ~1KB |
| `ospadep_audit_log` | Activity logs (last 1000 records) | ~2MB |
| `ospadep_temp_quotation` | Current quotation state | ~5KB |

## Performance Considerations

- **Client-side calculation**: All pricing computed in browser for instant results
- **External data loading**: Prices loaded once at page load from `precios-data.json`
- **LocalStorage optimization**: Audit log limited to 1000 most recent entries
- **Lazy loading**: Plan details loaded only when viewing results

## Troubleshooting Common Issues

<AccordionGroup>
  <Accordion title="Prices not loading">
    Check that `precios-data.json` is accessible and valid JSON. Open browser console to see any loading errors.
  </Accordion>
  
  <Accordion title="Admin panel not accessible">
    Verify credentials in `auth-system.js:35-46`. Check localStorage for existing session that may need clearing.
  </Accordion>
  
  <Accordion title="Quotations showing incorrect prices">
    Ensure `precios-data.json` version matches expected month. Check browser console for calculation errors.
  </Accordion>
  
  <Accordion title="User sessions not persisting">
    Check browser localStorage is enabled and not full. Clear old data if approaching storage limits.
  </Accordion>
</AccordionGroup>

<Warning>
  **Production Deployment:** This system stores all data client-side. For production use, implement server-side authentication, database storage, and secure API endpoints.
</Warning>
