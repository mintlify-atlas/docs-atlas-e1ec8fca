## Descripción General

Las actualizaciones masivas permiten aplicar aumentos o descuentos porcentuales a múltiples planes simultáneamente, ahorrando tiempo y garantizando consistencia en los ajustes de precios.

<Info>
  Ideal para aumentos generales por inflación, ajustes estacionales o descuentos promocionales.
</Info>

## Actualización Masiva por Porcentaje

### Cuándo Usar Actualizaciones Masivas

**Casos de uso comunes:**
- **Ajustes por inflación**: Aplicar aumentos mensuales o trimestrales
- **Descuentos promocionales**: Reducir precios temporalmente
- **Actualización de tarifas**: Cuando el prestador modifica todos sus precios
- **Ajustes por categoría**: Modificar solo planes individuales o familiares

### Tipos de Actualización

  <Card title="Aumento Masivo" icon="arrow-trend-up">

## Aplicar Aumento Masivo

### Proceso de Aumento

<Steps>
  <Step title="Acceder a la función">
    En el panel administrativo, haz clic en **"Actualización Masiva"** y selecciona **"Aumento"**.
  </Step>
  
  <Step title="Ingresar porcentaje">
    Escribe el porcentaje de aumento deseado (por ejemplo, `1.5` para 1.5%).
    
    <Info>
      Los valores positivos incrementan el precio. No uses el símbolo `+`.
    </Info>
  </Step>
  
  <Step title="Seleccionar categoría">
    Elige qué planes actualizar:
    - **Todos los planes**: Afecta planes individuales y familiares
    - **Solo planes individuales**: Únicamente planes para una persona
    - **Solo planes familiares**: Únicamente planes para familias
  </Step>
  
  <Step title="Previsualizar cambios">
    El sistema muestra cuántos planes serán afectados y ejemplos de cálculos.
  </Step>
  
  <Step title="Confirmar operación">
    Confirma la acción en el modal de confirmación.
    
    <Warning>
      Esta acción modifica múltiples precios simultáneamente. Asegúrate de haber exportado un respaldo.
    </Warning>
  </Step>
</Steps>

### Ejemplo de Aumento del 1.5%

```javascript
// Cálculo automático del sistema
const porcentaje = 1.5;
const precioOriginal = 189278; // OMINT PLAN 4500

// Fórmula aplicada
const precioNuevo = Math.round(precioOriginal * (1 + porcentaje / 100));
// precioNuevo = 192119

// Cambio: +$2,841
```

### Resultado de la Operación

Al confirmar, el sistema:

1. Aplica el aumento a todos los planes seleccionados
2. Guarda los nuevos precios en `localStorage`
3. Registra la operación en el histórico
4. Muestra un mensaje de confirmación:

```text
✅ Se aplicó un aumento del 1.5% a 24 planes
```

## Aplicar Descuento Masivo

### Proceso de Descuento

<Steps>
  <Step title="Acceder a la función">
    En el panel administrativo, haz clic en **"Actualización Masiva"** y selecciona **"Descuento"**.
  </Step>
  
  <Step title="Ingresar porcentaje">
    Escribe el porcentaje de descuento (valor positivo, por ejemplo, `10` para 10% de descuento).
    
    <Warning>
      El sistema interpreta este valor como descuento. No uses valores negativos.
    </Warning>
  </Step>
  
  <Step title="Seleccionar categoría">
    Elige qué planes actualizar (todos, individuales o familiares).
  </Step>
  
  <Step title="Confirmar operación">
    Revisa y confirma los cambios.
  </Step>
</Steps>

### Ejemplo de Descuento del 10%

```javascript
// Cálculo para descuento
const porcentaje = 10;
const precioOriginal = 189278; // OMINT PLAN 4500

// Fórmula aplicada
const precioNuevo = Math.round(precioOriginal * (1 - porcentaje / 100));
// precioNuevo = 170350

// Cambio: -$18,928
```

## Lógica de Cálculo

### Código de Aplicación

```javascript
// Función en script.js (línea ~4702)
function applyBulkIncrease() {
  const percentage = parseFloat(document.getElementById('increase-percentage').value);
  const category = document.getElementById('increase-category').value;
  
  // Validaciones
  if (isNaN(percentage) || percentage <= 0) {
    alert('Por favor, ingresa un porcentaje válido (mayor a 0)');
    return;
  }
  
  let updatedPlans = 0;
  const changes = [];
  
  // Aplicar cambios según categoría
  Object.keys(planesData).forEach(categoryKey => {
    if (category === 'all' || 
        (category === 'individual' && categoryKey === 'individual') ||
        (category === 'familiar' && categoryKey === 'familiar')) {
      
      planesData[categoryKey].forEach(plan => {
        const oldPrice = plan.price;
        const newPrice = Math.round(oldPrice * (1 + percentage / 100));
        plan.price = newPrice;
        updatedPlans++;
        
        changes.push({
          plan: plan.name,
          category: categoryKey,
          oldPrice: oldPrice,
          newPrice: newPrice,
          change: newPrice - oldPrice
        });
      });
    }
  });
  
  // Guardar cambios
  saveCustomPrices();
  
  // Registrar en histórico
  addToHistory({
    type: 'bulk_increase',
    description: `Aumento masivo del ${percentage}% aplicado`,
    plansAffected: updatedPlans,
    changes: changes
  });
}
```

## Operaciones Destructivas

<Warning>
  Las actualizaciones masivas son operaciones irreversibles que afectan múltiples registros simultáneamente.
</Warning>

### Precauciones Importantes

1. **Exporta siempre antes de actualizar**
   ```bash
   Panel Admin > Exportar Configuración
   # Guarda: ospadep_config_2025-03-05.json
   ```

2. **Verifica el porcentaje dos veces**
   - Un error de tipeo puede causar cambios significativos
   - `15` en lugar de `1.5` = 15% en lugar de 1.5%

3. **Revisa la categoría seleccionada**
   - Asegúrate de seleccionar la categoría correcta
   - "Todos" afecta a TODOS los planes

4. **Confirma en el modal**
   - Lee cuidadosamente el mensaje de confirmación
   - Verifica la cantidad de planes afectados

### Recuperación de Errores

Si aplicaste cambios incorrectos:

<Steps>
  <Step title="Opción 1: Restaurar precios originales">
    ```text
    Panel Admin > Restaurar Precios Originales
    ```
    Esto revierte TODOS los cambios a los valores iniciales del sistema.
  </Step>
  
  <Step title="Opción 2: Importar respaldo">
    ```text
    Panel Admin > Importar Configuración
    Selecciona: ospadep_config_[fecha].json
    ```
    Restaura los precios desde un archivo exportado anteriormente.
  </Step>
  
  <Step title="Opción 3: Aplicar corrección inversa">
    Si aplicaste +10% por error, aplica -9.09% para compensar:
    ```javascript
    // Fórmula de corrección
    porcentajeCorreccion = -((porcentajeError / (100 + porcentajeError)) * 100)
    
    // Ejemplo: error +10%
    // corrección = -((10 / 110) * 100) = -9.09%
    ```
  </Step>
</Steps>
